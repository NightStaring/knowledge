# 基础概念

> 认证与鉴权的核心概念、术语、技术基础。

---

## 1. 认证 vs 鉴权

这是最容易被混淆的两个概念：

| 概念 | 英文 | 含义 | 类比 |
|------|------|------|------|
| **认证** | Authentication (AuthN) | **你是谁？** 验证用户身份 | 进公司刷工卡 |
| **鉴权** | Authorization (AuthZ) | **你能做什么？** 验证用户权限 | 刷卡后哪些楼层能去 |

**认证在前，鉴权在后**：先确定身份，再判断权限。

## 2. HTTP 的无状态性

HTTP 本身是**无状态**的——每次请求都是独立的，服务器不知道前后请求是否是同一个人。

```
请求1: GET /profile
  → 服务器: "你是谁？我不认识你"

请求2: GET /profile (带 Cookie: session=abc123)
  → 服务器: "哦，session=abc123 的用户，这是你的资料"
```

**登录保持要解决的问题**：在无状态的 HTTP 之上，建立有状态的会话。

## 3. 哈希与加密

### 3.1 哈希（Hash）

- **单向**：不可逆，不能从哈希值还原原文
- **特点**：相同输入 → 相同输出
- **用途**：密码存储、数据完整性校验

```text
"mypassword" → SHA-256 → "89e01536ac6e287a..."
```

### 3.2 加密（Encryption）

- **双向**：可加密也可解密
- **对称加密**：加密解密用同一个密钥（AES）
- **非对称加密**：公钥加密，私钥解密（RSA）
- **用途**：数据传输加密、JWT 签名

### 3.3 常见哈希算法

| 算法 | 用途 | 安全性 |
|------|------|--------|
| **bcrypt** | 密码存储 | ✅✅ 推荐（内置 salt + 可调 cost） |
| **scrypt** | 密码存储 | ✅✅ 推荐（内存密集型，抗 GPU） |
| **argon2** | 密码存储 | ✅✅✅ 当前最推荐 |
| SHA-256 | 数据完整性 | ✅ 但**不适合**密码存储 |
| MD5 | — | ❌ 已破解，禁止使用 |

## 4. Base64 与 JWT

Base64 **不是加密**，是编码。JWT 的三段式是 `Header.Payload.Signature`：

```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjF9.sM3fL9wX8A...
├── Header (Base64) ────── 算法信息
├── Payload (Base64) ───── 数据（可读！不要放密码）
└── Signature ──────────── 防篡改签名
```

**关键**：Payload 只是 Base64 编码，**不是加密**，任何人都可以解码看到内容。所以**不要在 JWT 里放密码等敏感信息**。

## 5. 术语速查

| 术语 | 说明 |
|------|------|
| **Session** | 服务端存储的用户会话数据 |
| **Cookie** | 浏览器存储的小数据片段，每次请求自动带上 |
| **Token** | 令牌，代表用户身份的凭证 |
| **JWT** | JSON Web Token，一种自包含的 Token 格式 |
| **OAuth** | 开放授权协议，允许第三方应用获取有限访问权限 |
| **SSO** | 单点登录，一次登录访问多个系统 |
| **RBAC** | 基于角色的访问控制 |
| **ABAC** | 基于属性的访问控制 |
| **CSRF** | 跨站请求伪造 |
| **XSS** | 跨站脚本攻击 |
| **2FA** | 双因素认证 |
| **MFA** | 多因素认证 |
