# AI 辅助编程

> Claude Code 工作流、Prompt 工程、AI 协作模式。

---

## 1. AI 编程工作流

### 1.1 推荐流程

```
1. 需求分析
   ↓
2. /plan 进入计划模式
   AI 分析需求、设计方案、识别风险
   ↓
3. 确认计划
   你审查计划，确认后执行
   ↓
4. 分步实现
   让 AI 逐个功能实现
   ↓
5. 代码审查
   /code-review 审查代码
   ↓
6. 测试验证
   运行测试，验证功能
```

### 1.2 什么适合 AI

| 适合 AI | 不适合 AI |
|---------|-----------|
| 生成样板代码 | 架构决策 |
| 编写单元测试 | 安全敏感逻辑 |
| CRUD 接口实现 | 核心算法设计 |
| Bug 排查 | 复杂业务规则 |
| 代码重构 | 技术选型 |
| 文档编写 | 代码审查（仍需人工） |

### 1.3 什么情况重新开对话

```
✅ 应该开新会话：
  - 切换到完全不同的任务
  - 上下文太长影响效果时
  - 换项目时

❌ 不需要开新会话：
  - 同一任务的不同步骤
  - 当前对话中的子任务
  - 简单的追问
```

## 2. Prompt 工程

### 2.1 好 Prompt 的原则

```
❌ 模糊：
  "帮我写个登录"

✅ 清晰：
  "用 Spring Boot 3 + JWT 实现登录接口，
   需要用户名密码验证，返回 access token 和 refresh token，
   token 有效期 30 分钟，密码用 BCrypt 加密"
```

**好 Prompt 四要素：**
1. **上下文** — 用什么技术栈、在什么项目中
2. **目标** — 要做什么
3. **约束** — 有什么限制（性能、安全、兼容性）
4. **输出格式** — 期望的结果形式

### 2.2 场景示例

```bash
# 让 AI 生成代码时带教学注释
"请生成订单查询接口，关键步骤加中文注释，说明为什么这么做"

# 让 AI 分析问题
"这段代码有什么性能问题？如何优化？列出 3 个改进点"

# 让 AI 做知识问答
"解释一下 Spring @Transactional 的 REQUIRES_NEW 和 NESTED 的区别，
 给一个代码示例"
```

## 3. 高效协作模式

### 3.1 用 Skill 规范流程

```
每次做重要操作前，问自己：
"有没有 Skill 可以用？"

开发新功能 → brainstorming
发现 Bug → systematic-debugging
写完代码 → requesting-code-review
不确定方案 → Plan 模式
```

### 3.2 多轮迭代

```
第 1 轮: "生成订单模块的 CRUD"
第 2 轮: "加上参数校验和异常处理"
第 3 轮: "加上事务控制和并发锁"
第 4 轮: "/code-review"

不要一次让 AI 做完所有事
分步迭代效果更好
```

### 3.3 善用 CLAUDE.md

```
项目根目录的 CLAUDE.md 能让 AI 了解项目上下文：

- 技术栈（Spring Boot 3、MyBatis-Plus、MySQL）
- 项目规范（代码风格、commit 规范）
- 架构说明（包结构、模块划分）
- 常用命令（mvn clean install、docker-compose up）
```

## 4. 局限性

```
知道 AI 的边界，避免浪费时间：

✅ AI 能做的：
  - 写你知道怎么写的代码（加速）
  - 查你记得但不确切的 API
  - 生成测试数据和测试用例
  - 解释代码逻辑

❌ AI 做不好的：
  - 需要深层业务理解的需求
  - 涉及机密/敏感数据的处理
  - 需要物理直觉的判断
  - 没有明确对错的架构决策

⚠️ 需要人工确认的：
  - 生成的代码是否安全（SQL 注入、XSS）
  - 依赖版本是否最新
  - 是否符合项目编码规范
  - 边界条件是否覆盖
```
